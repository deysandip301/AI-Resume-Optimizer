# Contributing Guide

Thank you for your interest in contributing!

## How to Contribute
1. Fork the repository
2. Create a new feature branch
3. Make your changes with clear, descriptive commit messages
4. Ensure all tests and linters pass locally
5. Open a pull request (PR) to the main branch

## Code Style
- Follow PEP8 for Python code
- Use descriptive variable and function names
- Keep functions and modules focused and concise

## Testing
- Add or update tests in the `tests/` directory
- Run `pytest --cov=src --cov-report=term-missing` before submitting

## Pull Request Checklist
- [ ] Code builds and passes all CI checks
- [ ] Tests are added/updated and passing
- [ ] Documentation is updated if needed
- [ ] No sensitive data or secrets are committed

## Communication
- Use clear, respectful language in issues and PRs
- For major changes, open an issue first to discuss

We appreciate your contributions!
